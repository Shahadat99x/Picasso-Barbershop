# Design System

## Design Direction
The visual direction is:
- premium
- elegant
- minimal
- modern beauty
- mobile-first
- clean and conversion-focused

Recommended tone:
minimal luxury with soft editorial feel

## UX Priorities
- strong first impression
- easy booking path
- excellent mobile usability
- clean branch discovery
- clean service discovery
- polished gallery presentation
- bilingual usability without clutter

## Visual Principles
- generous whitespace
- strong typography hierarchy
- subtle motion only
- clear CTA contrast
- premium imagery
- restrained card styles
- consistent spacing

## Color Direction
Use a neutral luxury palette.
Suggested direction:
- soft off-white background
- warm beige / sand accent
- charcoal / deep neutral for text
- muted gold or rose-toned accent only if tasteful
- avoid loud saturated colors

Exact tokens should be finalized during implementation.

## Typography
Use a polished combination:
- elegant display heading style
- clean readable body text
- high legibility on mobile
- strong heading hierarchy

Rules:
- headings should feel premium
- body copy should remain simple and readable
- avoid decorative overload

## Component Principles
Build reusable components for:
- header
- mobile nav
- footer
- CTA button
- secondary button
- section heading
- service card
- branch card
- testimonial card
- gallery card
- blog card
- specialist card
- promo banner
- contact block
- breadcrumb

## Card Rules
Cards should:
- feel light and premium
- not be overly shadow-heavy
- use rounded corners consistently
- maintain image quality
- remain readable on small screens

## Button Rules
Primary CTA:
- should stand out
- must be used consistently for booking

Secondary CTA:
- calmer style
- used for discovery or secondary actions

Mobile rule:
- sticky booking CTA should be available where useful

## Layout Rules
- mobile-first
- container widths should be consistent
- spacing must scale smoothly
- sections should not feel cramped
- large visual sections should still be performant

## Responsive Rules
Must support:
- small phones
- modern mobile widths
- tablet
- desktop
- large desktop

Key mobile priorities:
- thumb-friendly spacing
- readable text
- tap targets
- easy menu behavior
- clear booking CTA

## Motion Rules
Use motion sparingly.
Allowed:
- fade in
- slight card motion
- subtle hover behavior
- elegant page transitions only if lightweight

Avoid:
- flashy interactions
- heavy animation
- anything that harms performance

## Form Rules
Forms must be:
- simple
- clear
- accessible
- low-friction
- mobile-friendly

## Image Rules
Images are central to the brand.
Must ensure:
- premium crop quality
- responsive delivery
- consistent ratios where possible
- strong gallery experience
- alt text support

## Accessibility Rules
- clear contrast
- visible focus states
- keyboard support
- readable text sizes
- accessible forms
- reduced motion support later

## Design Anti-Patterns
Do not:
- overload the homepage
- use too many accent colors
- create an overly corporate dashboard look for public pages
- use generic SaaS UI patterns on beauty pages
- overuse gradients, glows, or loud borders
