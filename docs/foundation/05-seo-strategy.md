# SEO Strategy

## SEO Goals
- rank for relevant salon searches in Vilnius
- rank for branch-specific queries
- rank for service-specific queries
- support Lithuanian primary search intent
- support English-speaking user discovery in Vilnius
- achieve strong mobile technical performance

## Core SEO Principles
- every important service gets its own page
- every branch gets its own page
- all public pages need unique metadata
- content should be useful, not thin
- internal links should be intentional
- image optimization is mandatory
- bilingual SEO must be structurally correct

## Priority SEO Targets
### Local SEO
- salon in Vilnius
- beauty salon in Vilnius
- service + Vilnius
- branch/location-focused search intent

### Service SEO
Examples:
- haircut
- hair coloring
- nails
- keratin
- brows
- other real services depending on business offering

### Informational SEO
Use blog content for:
- beauty tips
- seasonal care
- hair trends
- treatment comparisons
- local beauty advice

## URL Strategy
Lithuanian:
- `/`
- `/paslaugos/[slug]`
- `/filialai/[slug]`
- `/blogas/[slug]`

English:
- `/en`
- `/en/services/[slug]`
- `/en/branches/[slug]`
- `/en/blog/[slug]`

## Metadata Rules
Every public page should define:
- title
- meta description
- canonical
- OG image where relevant

Do not reuse generic metadata across all pages.

## Service Page SEO Rules
Each important service page must include:
- unique title
- unique description
- useful body copy
- FAQ where helpful
- related branch links
- clear booking CTA
- image alt text

## Branch Page SEO Rules
Each branch page must include:
- unique location-specific intro
- address
- hours
- phone
- map
- services available there
- unique local content
- transport / parking notes where useful

Do not duplicate the same text across branch pages.

## Blog SEO Rules
Each blog article should include:
- unique title
- unique excerpt
- internal links to service or branch pages
- useful content with real value
- strong cover image
- structured headings

## Internal Linking Rules
Must link naturally between:
- homepage -> services
- homepage -> branches
- services -> related branches
- branches -> available services
- blog -> services/branches
- services -> booking

## Structured Data
Planned structured data:
- LocalBusiness
- BreadcrumbList
- FAQPage where relevant
- Article for blog posts

## Sitemap
Must include:
- homepage
- service pages
- branch pages
- blog pages
- other important public pages

Must exclude:
- admin routes
- draft content
- private routes

## Robots Rules
Must allow indexing of public content.
Must block:
- admin
- private content
- irrelevant internal paths

## Image SEO Rules
All important images should have:
- descriptive alt text
- optimized dimensions
- compressed delivery
- appropriate filenames if controllable

## Bilingual SEO Rules
- Lithuanian is primary
- English pages must have equivalent value, not empty placeholders
- hreflang should be implemented during localization phase
- canonical logic must avoid duplication issues

## Performance Rules
SEO is tightly linked to performance.
Must prioritize:
- fast LCP
- optimized images
- reduced unnecessary client rendering
- mobile-friendly UX
- stable layout

## Content Rules
Avoid:
- thin pages
- duplicated location copy
- untranslated mixed pages
- filler content
- keyword stuffing

Prefer:
- clear user-first copy
- branch-specific detail
- service-specific clarity
- trust-building content
